How To Properly Customize Your Templatetop

Out templates/themes offer you the possibility to customize them without altering any of the main CSS files. This is very important if you want to keep your template/theme updated.

Each template/theme includes a CSS file named "custom.css" which is located in "css/custom.css". You can add your custom CSS rules there, and when an update will be released you will be able to update your template/theme and keep your custom CSS rules.

This template includes a lot of useful widgets and elements. To see them all please open the CSS file called "css/elements.css". There you will see a long list of available CSS classes for buttons, icons and other elements.

Working With Skinstop

Some of our templates may have multiple color styles, or skins.

If you open the CSS file called "skin.css" which can be found in "css/skin.css", you will see a few lines of code that look like the ones bellow:

/*@import url('skins/orange.css');*/
/*@import url('skins/red.css');*/
/*@import url('skins/pink.css');*/
/*@import url('skins/green.css');*/
/*@import url('skins/blue.css');*/

If you want to activate a skin, just remove the comments surrounding the line you wish to activate, and delete/comment the lines you do not need. Leave only one line of code active, for the skin you want.

In some cases not all elements are altered by the skin, that is because some elements are flexible and accept various classes to customize them. For example a button can have a class that changes its color, so the skin is not needed to customize that element, and all you need to do is add an extra class.

HiDPI/Retina Displaytop

To make this template look good on a HiDPI display all of its icons and and assets have been optimized. You will notice that each icon has two sizes, the normal size and a larger size (double the size of the normal size).

All the main CSS rules for HiDPI are in "css/hidpi.css". Some can be found in the skin files.

You can turn off this feature by removing the "hidpi.css" file from all the pages, or by deleting/commenting out all the rules in that file.

Configuring The Contact Formtop

A working contact form is included with all our templates.

Before you can use the contact form you have to configure it. You will find a file named "contact.php" in folder "php/contact.php" and you will have to edit it.

Open that file in any code editor, and locate a line that looks like the code bellow:

$emailAddress = 'mail@example.com';

Replace that email address with your own and the contact form will work.

Our templates use a flexible validation system that allows you to add more fields without making any javascript changes. You will have to edit the file named "contact.php" and also the contact page.

Portfolio Pagestop

Usually our templates include 4 portfolio pages.

There is no limit to how many items a portfolio page can have.

To add more items to a page, simply duplicate the block of code that makes up an item (they are all commented), and add your new images.

The "data-type" attribute for each item needs to have at least one of the "data-type" attribute values the menu items have. You can set multiple values for each item's "data-type" attribute using commas, but be careful not to include any whitespaces.

Upload Your Templatetop
